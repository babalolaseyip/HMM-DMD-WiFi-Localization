Key Features

üî¨ DMD Feature Extraction

Dynamic Mode Decomposition reduces RSSI dimensionality from 14 APs to 7 features while preserving spatial-temporal information.

üîÑ HMM Classification

Gaussian Mixture Hidden Markov Models with k-means initialization and Baum-Welch EM algorithm for robust localization.

üìä Spatial-Temporal Modeling

Captures both spatial correlation across APs and temporal correlation of RSSI measurements over time.

‚ö° Low Computational Cost

Processing time of only 0.00016s per test sample, suitable for real-time applications.

üéØ High Accuracy

Achieves 94.65% overall accuracy with mean localization error of ~1.69m on CRAWDAD dataset.

üîß Robust to Noise

DMD decomposition effectively handles multipath interference and environmental variations.

# Clone the repository git clone https://github.com/babalolaseyip/HMM-DMD-WiFi-Localization.git cd HMM-DMD-WiFi-Localization # Install dependencies pip install -r requirements/base.txt # Install package in development mode pip install -e .

Basic Usage

# Train with synthetic data python scripts/train_model.py --use-synthetic --output-dir results # Run tests pytest -v # Train with custom config python scripts/train_model.py --config config/default.yaml

Python API Example

CRAWDAD Dataset

The implementation uses the CRAWDAD Mannheim/Compass dataset from University of Mannheim:

  • Test area: 312 m¬≤ office building floor
  • Access Points: 14 APs (802.11)
  • Reference Points: 612 grids with 0.5m spacing
  • Training samples: 110 RSSI samples per RP (72,600 total)
  • Test samples: 110 samples at 83 random coordinates (9,460 total)

Viterbi Decoding

Online Stage:

  1. Extract DMD features from test RSSI

  5. Return estimated user location
